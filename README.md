# Machine Learning Projects

这是我的机器学习课程项目合集。目录按学习章节组织，每一章下面放对应的实践案例。

这个仓库的目标是让“学过的知识点”和“写过的代码案例”保持一致：先按章节理解概念，再通过项目复现流程，最后把可运行代码沉淀下来。

## 学习路线

| 章节 | 主题 | 当前项目 |
| --- | --- | --- |
| `01_ml-overview` | 机器学习概述 | 暂无独立项目 |
| `02_knn` | KNN 算法 | `iris-knn-classifier`、`digits-knn-classifier` |
| `03_linear-regression` | 线性回归 | `linear-regression-from-scratch` |
| `04_logistic-regression` | 逻辑回归 | 暂无独立项目 |
| `90_pending-review` | 待归类/待复盘 | `spam-classifier` |

## 章节说明

- `01_ml-overview`：机器学习整体框架和基础概念，目前主要作为知识背景。
- `02_knn`：KNN 分类流程、距离度量、特征预处理、交叉验证和网格搜索。
- `03_linear-regression`：线性回归、损失函数、梯度下降、拟合效果和评估指标。
- `04_logistic-regression`：逻辑回归、分类评估、混淆矩阵、Precision、Recall、F1、ROC 和 AUC。
- `90_pending-review`：已经完成但还没有归入当前章节体系的机器学习相关项目。

## 目录原则

- 只创建已经学到的章节目录。
- 每个章节目录的 `README.md` 说明本章学习目标和案例入口。
- 每个具体项目自己的 `README.md` 说明运行方式、功能和学习收获。
- 后续学到新章节时，再新增 `05_xxx` 这样的章节目录。

## GitHub 维护方式

这个目录对应 GitHub 仓库：`machine-learning-projects`。

历史上部分案例曾经是独立仓库；之后统一收进这个领域仓库，方便按机器学习章节维护。每个具体案例的详细项目说明会保留在项目自己的 `README.md` 中。
